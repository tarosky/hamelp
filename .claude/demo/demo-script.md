# Jovian デモ実演スクリプト（台本）

PubPla AI Help Center の AI Overview を、架空サービス「Jovian」のFAQで実演するための台本。
狙いは **「AI が FAQ に接地（grounding）して答えている」＝事前知識の丸写しではない** ことを、
観客に体感させること。詳細設定は [[Universe.md]]（.claude/Universe.md）参照。

## 事前準備

- トップページ（または「ヘルプセンター」ページ）の **AI Overview ブロック**を開いておく。
- AI Overview の **モードは「Conversation（多ターン）」**（手順4のマルチターンで必要）。
- 余裕があれば、隣に ChatGPT 等の汎用チャットを開いておくと「種明かし」が刺さる。

## 実演の流れ（この順番で聞く）

escalation 設計：ワンダーで掴み → 接地で驚かせ → 罠で念押し → マルチターンで締め → ネガティブで信頼を残す。

### ① つかみ（ワンダー）
- **質問**: 「この商品の材料はどこから来るんですか？」
- **AIの答え**: 木星の雲頂に漂う生物「ネフェレ」を採取艇「ヘーベ」が会合期（約398日ごと）に採取…
- **トーク**: 「世界観がちゃんと出てますね。これ全部、皆さんが書いたFAQから喋ってます」

### ② 接地証明（ここが本丸）
- **質問**: 「若返りの施術を受けたあと、MRI検査は受けられますか？」
- **AIの答え**: 「**投与後72時間はMRI禁止**。テロメリンが磁性を帯びるため」＋ソース引用
- **トーク（種明かし）**: 「"72時間""磁性"——こんな情報、**ChatGPTに同じ質問をしても絶対に出てきません**。実在しない製品ですから。AIは皆さんのFAQを"読んで"答えているんです。引用元リンクも付くので、根拠も追えます」
  - ※汎用チャットを開いているなら、同じ質問を投げて「知らない／一般論」しか返らないのを見せると最高。

### ③ 反直感の罠（念押し）
- **質問**: 「早く若返りたいので、毎日飲んでもいいですか？」
- **AIの答え**: 「**重複使用は厳禁。"老化リバウンド"で逆効果。21日に1本を守って**」
- **トーク**: 「常識なら"多く飲むほど効きそう"。でもAIは勝手に推測せず、FAQの指示どおり"逆効果"と正しく止めます。間違った自己判断を防げる＝サポート品質が上がる」

### ④ マルチターン＋合成（フィナーレ）
会話モードのまま、続けて質問する。
- **質問1**: 「解約したいです」 → 手順と締切（次回お届けの14日前 12:00）
- **質問2**: 「その締切を過ぎたらどうなりますか？」 → 次回便は発送・課金、解約は次々回便から
- **質問3**: 「返金はされますか？」 → 日割りなし・未発送分はネビュラ・ポイント充当
- **トーク**: 「"その""返金は"と、前の文脈を覚えたまま会話が続きます。しかも回答は複数のFAQ記事をまたいで合成されている。問い合わせフォームに来る前に、ここで自己解決できる」

### ⑤ ネガティブ確認（信頼を残して締める）
- **質問**: 「火星にも配送してもらえますか？」
- **AIの答え**: 「**FAQに記載がありません**」（＋分かる範囲の隣接情報）
- **トーク**: 「知らないことは"知らない"と言う。デタラメを作らない。これが業務で使える一番の条件です。"それっぽい嘘"を返すAIは怖くて使えませんから」

## 締めのピッチ（1行）

「会員登録もチャットボット構築も不要。**FAQを書くだけ**で、24時間"中身を分かっている"サポートが立ち上がります」

## 補足・トラブル時

- 回答が古い/出ない場合：FAQを保存し直すとカタログが再構築される（数十秒）。`wp hamelp rebuild` でも可。
- レート制限（既定 5回/IP）に注意。リハーサルで使い切らないこと。
- シードの再投入は `.claude/demo/publish.sh`（要 direnv 環境変数）。
